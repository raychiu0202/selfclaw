# Selfclaw - 终端命令功能设计文档

## 1. 功能概述

**核心需求：** 用户在对话中表达操作意图（如"创建文件"、"查看目录"），AI自动识别并执行相应终端命令，将执行结果作为回复的一部分返回。

**设计理念：** 用户无感知、极简设计、智能识别、动态显示

**系统定位：** 为 Selfclaw AI Agent 提供"手和脚"，实现从"对话"到"行动"的能力扩展

**设计灵感：** 仿照OpenClaw的交互体验和功能设计

## 1.1 极简设计原则

为确保系统稳定性和用户体验，本项目严格遵循以下极简设计原则：

1. **功能最小化**
   - 仅保留核心、必需的功能
   - 避免过度设计和复杂化
   - 每个功能都有明确的使用场景

2. **用户无感知**
   - 不需要独立的终端入口
   - AI自动识别意图
   - 自动执行并返回结果

3. **界面简洁性**
   - 保持纯对话界面
   - 命令执行融入对话流
   - 直观的用户操作流程

4. **代码精简性**
   - 最小化代码复杂度
   - 避免不必要的依赖和抽象
   - 保持代码可读性和可维护性

5. **性能优先**
   - 快速响应，低延迟
   - 最小化资源占用
   - 优化关键路径

6. **安全性优先**
   - 白名单机制
   - 严格的权限控制
   - 简单可验证的安全策略

7. **用户体验优先**
   - 仿OpenClaw的流畅交互
   - 实时动态显示执行过程
   - 清晰的视觉反馈

## 2. 安全命令白名单

### 2.1 允许的命令列表

#### 只读命令
| 命令 | 参数限制 | 说明 | 触发关键词 |
|------|---------|------|-----------|
| `ls` | -la, -l, -h | 列出文件和目录 | 查看、列出、目录、文件 |
| `pwd` | 无参数 | 显示当前工作目录 | 当前目录、当前位置 |
| `cat` | 仅限项目目录 | 查看文件内容 | 查看、读取、显示文件 |
| `grep` | -n, -i, 限制搜索范围 | 搜索文件内容 | 搜索、查找、包含 |
| `head` | -n, 仅数字参数 | 查看文件前N行 | 前几行、头部 |
| `tail` | -n, 仅数字参数 | 查看文件后N行 | 后几行、尾部 |
| `wc` | -l, -w, -c | 统计文件行/词/字数 | 统计、行数、字数 |
| `date` | 无参数 | 显示当前时间 | 时间、日期 |

#### 写操作命令（严格限制）
| 命令 | 参数限制 | 说明 | 触发关键词 |
|------|---------|------|-----------|
| `touch` | 仅限项目目录 | 创建空文件 | 创建文件、新建文件 |
| `mkdir` | 仅限项目目录, -p | 创建目录 | 创建目录、新建文件夹 |
| `echo` | 仅文本, 仅追加到文件 | 写入文件内容 | 写入、添加内容 |
| `rm` | -r, 仅限项目目录 | 删除文件/目录 | 删除文件、移除 |
| `mv` | 仅限项目目录内 | 移动/重命名 | 移动、重命名 |

### 2.2 命令执行流程

```
用户输入："帮我在当前目录创建一个test.txt文件"
    ↓
AI识别意图：创建文件
    ↓
AI生成命令：touch test.txt
    ↓
后端验证命令安全性
    ↓
实时推送命令状态：🔧 正在执行命令: `touch test.txt`
    ↓
执行命令并获取结果
    ↓
推送执行结果：✅ 命令执行成功: `touch test.txt`
    ↓
AI整合结果并回复："已创建文件 test.txt"
```

### 2.3 安全限制

- 路径限制：仅限项目目录，禁止 `..` 和绝对路径
- 文件名限制：只允许字母、数字、下划线、点、短横线
- 命令频率限制：每分钟最多执行10次命令
- 输出长度限制：最多10,000字符
- 禁止危险操作：`sudo`, `rm -rf /` 等

## 3. 动态命令执行显示（仿OpenClaw）

### 3.1 实时状态推送

Selfclaw实现了与OpenClaw类似的动态命令执行显示：

#### 命令开始状态
```json
{
  "type": "command_start",
  "command": "touch test.txt",
  "message": "🔧 正在执行命令: `touch test.txt`"
}
```

#### 命令成功状态
```json
{
  "type": "command_success",
  "command": "touch test.txt",
  "output": "",
  "message": "✅ 命令执行成功: `touch test.txt`"

```

```json
{
  "type": "command_error",
  "command": "rm /protected/file",
  "error": "Permission denied",
  "message": "❌ 命令执行失败: `rm /protected/file`"
}
```

### 3.3 用户体验特性

## 5. 安全机制

### 5.1 命令验证流程

```
用户输入命令
    ↓
格式验证（是否为合法命令）
    ↓
白名单检查（命令是否在白名单中）
    ↓
参数验证（参数是否合法）
    ↓
路径检查（防止路径遍历）
    ↓
执行命令
    ↓
实时推送执行状态
```

### 5.2 安全限制

- 命令超时：默认30秒，最大60秒
- 并发限制：最多2个命令同时执行
- 输出限制：最多10,000字符
- 执行历史：保留最近100条记录
- 工作目录限制：仅限项目目录
- 路径验证：相对路径，禁止绝对路径和路径遍历

## 6. 数据模型

### 6.1 命令执行历史表

```sql
CREATE TABLE command_history (
    id INT PRIMARY KEY AUTO_INCREMENT,
    conversation_id INT,
    command VARCHAR(500),
    output TEXT,
    error TEXT,
    exit_code INT,
    execution_time FLOAT,
    created_at DATETIME,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);
```

4. 不要使用 cd 命令，因为命令已经在正确的工作目录中执行
5. 所有文件名都应该是简单的相对路径，如：test.md, filename.txt, data/

请分析用户的请求，如果需要执行文件操作，生成相应的命令。可用命令包括：
- 创建文件：echo "内容" > filename 或 touch filename

## 10. 错误处理

| 错误类型 | HTTP状态码 | 描述 | 处理方式 |
|---------|-----------|------|----------|
| 命令不在白名单 | 403 | 命令未授权执行 | 返回错误消息，不执行 |
| 命令格式错误 | 400 | 命令语法错误 | 返回错误消息，不执行 |
| 执行超时 | 408 | 命令执行超时 | 终止命令，返回超时信息 |
| 并发限制 | 429 | 同时执行命令过多 | 返回限流信息，稍后重试 |
| 路径访问违规 | 403 | 尝试访问受限路径 | 返回路径错误，不执行 |
| JSON解析失败 | 200 | 流式数据解析失败 | 记录警告，继续处理 |
| undefined内容 | 200 | 收到无效内容 | 过滤无效内容，继续处理 |

## 11. 优化和修复

### 11.1 已修复的问题

#### 修复奇怪回复闪现问题
- **问题**：出现"Ahaaaaaahhhhhhha..."等奇怪回复然后消失
- **原因**：GLM API返回了不完整的数据，前端处理不当
- **解决**：
  - 后端添加内容过滤，过滤控制字符和异常内容
  - 前端改进JSON解析错误处理
  - 添加详细的错误日志

#### 修复内容消失问题
- **问题**：命令执行状态在对话完成后消失
- **原因**：前端在对话完成时清空了状态
- **解决**：移除状态清空逻辑，让命令状态保留显示

#### 改进错误处理
- **问题**：错误提示不够明确
- **原因**：错误信息不够详细
- **解决**：
  - 添加详细的错误消息
  - 改进错误提示格式
  - 添加错误分类处理

### 11.2 性能优化

- **流式传输优化**：使用`text/event-stream`媒体类型
- **缓存优化**：避免重复的数据库查询
- **日志优化**：添加详细的执行日志用于调试
- **前端优化**：改进状态管理，减少不必要的渲染
